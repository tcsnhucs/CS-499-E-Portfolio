Enhancement 3 - Databases

About This Artifact

For my third enhancement, I chose to enhance my final project in the CS 340: Advanced Programming Concepts course at SNHU. I created this project throughout the entire semester in CS-340. In this course we were tasked to create a Python program that allows the user to interface with a CSV data file, and view data from the file in a table, see a pie chart for data categorization, and see different data points on a map. For my enhancement of this artifact, I chose to refactor everything into Javascript code.

Why This Artifact

My main goal with this artifact was to showcase my ability to take a program built in one language, and replicate its functions in another language. The python file that the original artifact was based on already launched an html web interface to interact with the data, so this artifact made the most sense to me to showcase that ability, as I refactored the artifact into Javascript, a language focused on developing web apps. Beyond that, since the entire point of this artifact is to view a large database in a web view, it seemed like the obvious choice to pick for the category of databases.

Enhancements Made

I focused on keeping functions relatively the same between the refactored code and the original code. Because my goal with this artifact was to showcase my ability to refactor code from one programming language to another, I didn't want to make the new version significantly different from the original, so there isn’t much functionally different between the two versions of the artifact. However, the pie chart in the original had a glaring flaw, where there were hundreds of unique entries with 1 or 2 items that had that entry name visible on the pie chart. This made the pie chart practically unusable and so I fixed this by making a cut off point at 0.75% where every breed with less percentage occurrence than that shows up in a single “other” category. Besides that, I also added stylized CSS to the page which makes it much more user friendly and modern looking. The original artifact exclusively used HTML to display it's output, meaning my artifact appears much more modern and user-friendly.
